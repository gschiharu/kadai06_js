## ⑤工夫した点・こだわった点
初回ミーティングをして、すぐにトライアルをしたいというときに、すぐに通訳者のスケジュールがわかるように設定しました。
サービスとしては、３つあるので、それぞれの流れを考えて設定しました。

通訳選択→通訳者のスケジュール確認をする。
翻訳→COOPROのコンシェルジェとミーティング
プロジェクトマネージメント→COOPROのコンシェルジェとミーティング

## ⑥難しかった点・次回トライしたいこと(又は機能)

CEOがサービスを使用する際、１）サービス軸で選択　２）既に通訳が使いたいから、日時軸で選択の2通りがある。
今回は１）にて設計。
本当は２）で設計したくトライしました。
ですが、通訳者の詳細スケジュールをとりにいくAPI設定がCalendlyでうまくいかず、Googleカレンダーを調べてみようと思いましたが、今回は時間が間に合わなかったです。。

ですが、１）２）の両方でニーズがあるので、まずはこなせる１）の設定を今回行いました。

CEOがカレンダーで日時を選択する。
それに対して通訳者の日時をCalendlyのAPI（https://developer.calendly.com/api-docs/4b402d5ab3edd-calendly-developer）を設定し、空き時間の情報を取りに行く。

もしくは、GoogleカレンダーのAPI設定をして、空き時間をとりにいく方がシンプルかもしれないとTutorさんと話ました。CalendlyもGoogleカレンダーの情報をとりにいくので、オリジナルデータのAPIをつかった方がシンプルかもしれません。。

実際のアプリ開発のときまでにリサーチを続けておこうともいます！
